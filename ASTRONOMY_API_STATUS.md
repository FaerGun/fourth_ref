# Статус AstronomyAPI

## Текущая ситуация

**Ошибка 403 (Forbidden)** при запросе к AstronomyAPI означает, что доступ к API запрещен.

## Возможные причины

1. **Ключи API неверны или истекли**
   - Ключи в `docker-compose.yml` могут быть демо/тестовыми
   - Требуется регистрация на https://astronomyapi.com/ и получение новых ключей

2. **Endpoint недоступен на бесплатном тарифе**
   - `/api/v2/bodies/events` может требовать платную подписку
   - Бесплатный тариф может иметь ограничения

3. **Превышен лимит запросов**
   - API может ограничивать количество запросов в день/месяц

## Что делать

### Вариант 1: Получить валидные ключи (рекомендуется)
1. Зарегистрируйтесь на https://astronomyapi.com/
2. Получите `ASTRO_APP_ID` и `ASTRO_APP_SECRET`
3. Обновите в `docker-compose.yml`:
   ```yaml
   ASTRO_APP_ID: ваш_новый_id
   ASTRO_APP_SECRET: ваш_новый_secret
   ```
4. Перезапустите: `docker-compose restart php`

### Вариант 2: Использовать альтернативный endpoint
Проверьте документацию AstronomyAPI - возможно, есть другой endpoint для бесплатного тарифа.

### Вариант 3: Временная заглушка
Если API недоступен, можно показывать мок-данные или отключить секцию.

## Текущие ключи

Ключи установлены в `docker-compose.yml`, но могут быть:
- Демо/тестовыми (не работают)
- Истекшими
- Недостаточными для этого endpoint

## Вывод

**Да, это должно работать**, но для этого нужны:
1. ✅ Валидные ключи API (полученные после регистрации)
2. ✅ Доступный тарифный план (бесплатный или платный)
3. ✅ Правильный endpoint для вашего тарифа

**Сейчас ошибка 403 означает, что один из этих пунктов не выполнен.**

## Проверка

После получения новых ключей проверьте:
```bash
docker-compose logs php --tail 50 | grep -i astro
```

Если ключи валидны, но все еще 403 - возможно, нужен другой endpoint или платная подписка.
